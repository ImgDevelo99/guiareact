Paso 1: ¿Qué es React?

¡Bienvenido a esta guía interactiva de Fundamentos de React para principiantes! React es una biblioteca de JavaScript muy popular para construir interfaces de usuario (UI).

Desarrollada por Facebook (ahora Meta), permite crear componentes de UI reutilizables y eficientes. Su principal característica es que se enfoca en la "vista" de una aplicación web.

Conceptos Clave Iniciales:
  • Declarativo: Describes cómo debería lucir tu UI en cualquier estado, y React se encarga de actualizarla eficientemente cuando los datos cambian.
  • Basado en Componentes: Construyes aplicaciones con pequeñas piezas de código llamadas "componentes" que manejan su propio estado y pueden ser reutilizadas.
  • Aprende una vez, escribe en cualquier lugar: Puedes usar React para desarrollar aplicaciones web (React DOM), móviles (React Native), o incluso de escritorio.

En los siguientes pasos exploraremos cómo construir con estos conceptos.

Paso 2: JSX - JavaScript XML

JSX es una extensión de sintaxis para JavaScript que permite escribir código similar a HTML dentro de JavaScript. Aunque no es obligatorio, es la forma más común y recomendada de trabajar con React.

JSX facilita la descripción de cómo debe lucir la UI de un componente.

Demostración (Código):
// Sin JSX (menos común y más verboso) const elementNoJSX = React.createElement( 'h1', { className: 'greeting' }, '¡Hola, mundo!' ); // Con JSX (más legible y directo) const elementJSX = <h1 className="greeting">¡Hola, mundo!</h1>;
Demostración (Visualización Esperada):

¡Hola, mundo!

JSX parece HTML, pero es JavaScript. Puedes incrustar expresiones de JavaScript dentro de JSX usando llaves {}.

Incrustando JavaScript en JSX:
const name = 'SENA'; const greeting = <h1>¡Hola, {name}!</h1>; // Output esperado: ¡Hola, SENA!

Paso 3: Componentes Funcionales

La forma más común de definir un componente en React hoy en día es usar una función de JavaScript. Estos se llaman Componentes Funcionales.

Una función de componente funcional acepta un objeto props (propiedades) como argumento y retorna un elemento de React (generalmente escrito con JSX) que describe lo que debe aparecer en la pantalla.

Demostración (Código):
// Define un componente funcional simple function Saludo() { return <h1>¡Hola desde un componente!</h1>; } // Puedes usar el componente como si fuera una etiqueta HTML // <Saludo />
Demostración (Visualización Esperada):

¡Hola desde un componente!

Los nombres de los componentes siempre deben empezar con una letra mayúscula. Esto ayuda a React a distinguirlos de los elementos HTML estándar (como <div> o <span>).

Paso 4: Props (Propiedades)

Los componentes pueden recibir datos de sus componentes padres a través de props. Son como los atributos de los elementos HTML, pero puedes pasar cualquier tipo de dato JavaScript (strings, números, objetos, funciones, etc.).

Las props son de solo lectura. Un componente nunca debe modificar sus propias props.

Demostración (Código):
// Componente que recibe una prop 'nombre' function SaludoConNombre(props) { return <p>¡Hola, {props.nombre}!</p>; } // Uso del componente pasando la prop // <SaludoConNombre nombre="Aprendiz" /> // <SaludoConNombre nombre="Instructor" />
Demostración (Visualización Esperada):

¡Hola, Aprendiz!

¡Hola, Instructor!

Desestructuración de Props (Común):

Es muy común desestructurar el objeto props para acceder a las propiedades más fácilmente:

function SaludoConNombre({ nombre }) { // Desestructuración aquí return <p>¡Hola, {nombre}!</p>; }

Paso 5: State (Estado) con useState Hook

Mientras que las props reciben datos de afuera y son inmutables, el estado (state) es data privada del componente que puede cambiar a lo largo del tiempo y disparar re-renderizaciones.

En componentes funcionales, manejamos el estado usando Hooks, específicamente el hook useState.

useState es una función que retorna un array con dos elementos: el valor actual del estado y una función para actualizarlo. La llamas con el valor inicial del estado.

Demostración (Código - Contador Simple):
import React, { useState } from 'react'; // Necesitas importar useState function Contador() { // Declara una variable de estado 'count' y su función 'setCount' const [count, setCount] = useState(0); // 0 es el valor inicial return ( <div> <p>Hiciste clic {count} veces</p> <button onClick={() => setCount(count + 1)}> Haz Clic </button> </div> ); }
Demostración (Visualización Esperada & Comportamiento):

Hiciste clic 0 veces


{/* Imagina que al hacer clic, el número 0 cambia a 1, luego a 2, etc. */} {/* React re-renderiza el componente con el nuevo valor de 'count' */}

Cuando llamas a la función de actualización del estado (setCount en el ejemplo), React re-renderiza el componente con el nuevo valor del estado. Esto es fundamental para crear UIs dinámicas.

Paso 6: Renderizando Componentes

Para que tus componentes de React aparezcan en el navegador, necesitas "renderizarlos" en un elemento DOM real. Esto se hace típicamente una vez al inicio de tu aplicación.

Las versiones modernas de React usan ReactDOM.createRoot() y su método render().

Demostración (Código - Archivo de Entrada Principal, ej: index.js):
// Necesitas importar React y ReactDOM import React from 'react'; import ReactDOM from 'react-dom/client'; // Importa el componente principal de tu aplicación (ej: App) import App from './App'; // Asumiendo que App.js existe // Encuentra el elemento DOM donde quieres montar tu app (usualmente un div con id="root") const container = document.getElementById('root'); // Crea una raíz de React const root = ReactDOM.createRoot(container); // Renderiza tu componente principal dentro de la raíz root.render( <React.StrictMode> <App /> {/* Aquí usas tu componente <App> */} </React.StrictMode> );
Concepto Clave: El 'root' DOM

Generalmente, en tu archivo HTML (ej: index.html), tendrás un <div id="root"></div> vacío. React tomará control de este div y gestionará todo su contenido.

Demostración (Código - Archivo index.html):
<!DOCTYPE html> <html lang="es"> <head> <meta charset="utf-8" /> <title>Mi App React</title> </head> <body> <div id="root"></div> <!-- Aquí es donde React montará la app --> </body> </html>
